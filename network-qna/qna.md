### 빅데이터란? 
- 대용량 데이터를 뜻하며, 방대한 자료의 양을 가지고 있고 종류가 다양함
- 빅데이터 분석을 통해 비지니스에 궁극적인 가치를 창출할 수 있음

### 빅데이터 인프라 구축 시 고려 사항
- 데이터의 추출 시간이 중요하며 디시크의 성능이 좋아야 함

### 정형 데이터와 비정형 데이터의 차이
- 정형 데이터는 관계형 데이터베이스에 저장되도록 정제 및 구조화 됨
- 비정형 데이터는 포멧이나 체계가 없는 데이터

### 오픈스택과 가상화의 차이
- 오픈 스택은 클라우드 구축을 위한 플랫폼
- 가상화는 클라우드 구축을 위해 필요한 기술

### 광 구성이 가능한 라인업 장비
- Forti 80 이상, Axgate 200 이상 가능

### 와탭
- ai기반 트랜잭션 분석 모니터링 시스템

### KT프리미엄 VPN임대 장비
- V6424(국산) ME3400(외산)
  
### 장비 선정시 고려사항
- 고객의 회선속도랑 고객의 용도에 맞춰서 선정해야 함

### CC인증이란?
- Common Criteria
- 컴퓨터 보안을 위한 국제 표준 규격
- 국내와 국외가 존재
- 국가, 공공기관은 국내 CC, 나머지는 사업에 맞는 CC인증 필요
- 한국정보통신기술협회, 한국IT평가원, 한국인터넷진흥원

### 10 / 100 / 1000 Base-TX, 단위 및 차이점
- 모두 이더넷 케이블을 사용하여 데이터 통신을 지원한다
- 앞의 숫자는 대역폭을 의미한다
- 최대 전송속도, 대역폭 10Mbps, 100Mbps, 1000Mbps
- TX는 케이블의 송신 방식

### HTTP, HTTPS 차이
- http는 암호화나 인증이 필요없는 공개적인 데이터를 사용하는 경우 사용
- https는 SSL인증서를 http에 추가한 것

### BGP기능 포함 여부를 확인해야 하는 이유
- BGP는 Border Gateway Protocol로 
- 대규모 네트워크에서 라우팅 결정을 내리고 최적 경로를 찾기 위해 필요한 기능이기 떄문
- BGP기능이 있으면 다른 ISP와의 연결성을 향상하고 효율적으로 관리할 수 있기 때문
- 사설BGP vs 공인BGP
  
### 토폴로지란
- 네트워크에 연결된 컴퓨터와 케이블 및 구성요소의 배치도

### CDN
- 웹 콘텐츠를 손실없이 빠르고 안정적으로 대규모 사용자에게 전달하는 서비스

### L4 스위치란
- 4계층에서 동작하는 스위치
- 로드 밸런싱을 통한 네트워크 트래픽 관리
- 세션 유지, 패킷 필터링 기능 수행

### L3 스위치 라우터 차이
- L3스위치는 포워딩을 사용하여 라우팅을 수행하고 라우터는 소프트웨어적으로 처리를 함
- 따라서 라우터가 스위치에 비해 느리지만 더 많은 트래픽 처리량과 확장성이 높고 네트워크 보안 기능을 제공하기 때문에 보안적으로도 좋음
- L3스위치는 LAN환경에 최적화 되어 있고 특수 하드웨어 기반 포워딩을 통해 더 빠른 성능을 제공 함. 반면 라우터는 복잡한 네트워크에 더 적합하며 보안 및 확정성을 위한 고급 기능을 제공 함

### MTU란
- 네트워크에서 전송될 수 있는 최대 크기의 패킷 또는 프레임

### 네트워크 엔지니어가 뭐냐
- 네트워크 엔지니어는 기업의 컴퓨터 네트워크가 원활하게 동작하도록 관리하고, 문제가 발생했을 때 해결하는 역할을 함. 또한 네트워크의 보안을 유지하고, 새로운 기술을 적용하여 네트워크의 효율성과 성능을 개선하는 일도 담당 함

### TCP/UDP 차이
- TCP/IP는 신뢰성 있는 데이터 전송을 위해 설계된 프로토콜로, 높은 신뢰성을 보장하지만 UDP보다는 속도가 느리다. 3way 4way handshake를 통해 서버와 클라이언트가 통신하며, 혼잡제어와 흐름제아가 이루어 진다.
- UDP는 비연결성 프로토콜로 손상된 데이터에 대해서 재전송 하지 않고 신뢰성이 낮지만 TCP보다 빨라서 주로 스트리밍 서비스에 이용된다

### VLAN이란 무엇이고 왜 사용하는지 설명
- 물리적인 위치 상관 없이 로직으로 네트워크를 분할하는 기술을 뜻합니다.
- 네트워크의 트래픽을 관리하고, 보안을 강화하며, 네트워크 성능을 향상시킬 수 있습니다.

### 서브네팅이란
- 서브네팅은 하나의 네트워크를 여러게의 작은 네트워크로 분할하는 것을 말합니다.
- 이를 통해 IP주소를 효율적으로 관리할 수 있으며, 네트워크 트래픽을 줄이고 성능을 향상시킬 수 있습니다.

### DNS와 DHCP의 차이점을 설명
- DNS는 도메인 이름을 IP주소로 변환해 주는 시스템이다.
- DHCP는 네트워크에 연결된 장치에게 자동으로 IP를 할당하는 프로토콜 이다.
- DHCP는 컴퓨터에 자동으로 네임 서버 주소, IP주소, 게이트웨이 주소를 할당해

### 라우터와 스위치의 차이점
- 라우터는 네트워크 간의 통신을 가능하게 하는 장치로 IP주소를 기반으로 데이터를 전송한다.
- 스위치는 같은 네트워크 내의 장치들이 통신할 수 있또록 하는 장치로, MAC주소를 기반으로 데이터를 전송한다.

### IP주소와 MAC주소의 차이점
- 공인 IP주소는 인터넷 프로토콜로 할당되는 고유한 주소로 네트워크 상에서 컴퓨터를 식별하는데 사용 된다.
- MAC주소는 네트워크 인터페이스 컨트롤러(NIC)카드에 내장된 고유한 식별자로, 하드웨어를 식별하는 데 사용된다.

### VPN이란
- Virtual Private Network의 약자로 인터넷을 통해 사설 네트워크에 접속할 수 있게 해주는 기술이다.
- 원격 서버를 통해 트래픽의 경로를 변경하여 프로세스에서 트래픽을 암호화 하는 기술입니다.
- VPN은 원격에서 안전하게 사내 네트워크에 접속하거나, 인터넷 통신을 암호화 하여 개인정보를 보호하는데에 사용된다.

### 방화벽이란
- 네트워크 보안을 위해 정해진 규칙에 따라 접근을 제어하는 기술입니다.

### 로드 밸런싱이란
- 네트워크 트래픽을 여러 서버에 분산시켜 각 서버의 부하를 조절하는 기술 입니다.
